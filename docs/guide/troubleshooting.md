# 문제 해결

## `실행` 버튼을 눌렀는데 오류가 나옵니다

**가능한 원인**

- 코드 컴파일/런타임 에러
- Rust/Java 실행 시 Piston API 일시 장애 또는 요청 제한(HTTP 429 등)
- 확장 업데이트 이후 탭/확장 새로고침이 반영되지 않음

**확인 순서**

1. 확장 프로그램을 다시 로드하고 BOJ 탭을 새로고침
2. 다시 `실행` 후 출력 메시지(컴파일 에러/서버 오류) 확인
3. Rust/Java에서만 실패하면 잠시 후 재시도

## 제출이 실패합니다

**가능한 원인**

- BOJ 세션 만료
- Cloudflare 토큰(`cf-turnstile-response`) 만료
- 일시적인 네트워크/서버 문제

**해결**

1. 페이지 새로고침 후 다시 시도
2. BOJ 로그인 상태 재확인
3. 동일 문제 반복 시 브라우저 콘솔 에러 확인

## 원하는 언어가 선택 목록에 보이지 않습니다

확장은 현재 다음 BOJ 언어만 지원합니다.

- `84`(C++17), `95`(C++20), `93`(Java 11), `28`(Python 3), `116`(Rust 2024)

문제별로 BOJ 제출 폼에 노출된 언어와 동기화되므로, 위 언어라도 문제 조건에 따라 숨겨질 수 있습니다.

**해결**

1. 문제의 BOJ 기본 제출 폼에서 실제 제출 가능 언어를 먼저 확인
2. 확장 화면 상단의 `기본 BOJ 제출 폼으로 전환`을 눌러 원본 폼으로 제출
3. 확장 재로드 후 BOJ 탭 새로고침

## 수정 제출에서 코드가 비어 보입니다

확장은 `/submit/<problemId>/<submissionId>` 경로와 query/form/source 링크를 통해 원본 코드를 복원합니다.

복원이 늦는 경우가 있어 일정 시간 재시도를 수행하므로, 페이지 진입 직후 즉시 빈 코드여도 잠시 대기해 주세요.

## 테마/코드/테스트 케이스가 유지되지 않습니다

저장 데이터는 `chrome.storage.local`을 사용합니다.

- 브라우저 데이터 정리 정책
- 확장 재설치
- 동일 프로필 미사용

위 조건에서는 데이터가 유지되지 않을 수 있습니다.
